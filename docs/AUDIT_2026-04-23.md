## 4. CURRENT IMPLEMENTATIONS

### A. PROCTORING (Anti-Cheating)

**Implemented Features:**

1. **Face Detection**
   - Uses: MediaPipe (primary) + OpenCV (fallback)
   - Detects: faces_count, face_box, face_signature
   - File: `utils/proctoring_cv.py`

2. **Shoulder Visibility**
   - Detects upper body for proof of presence
   - Uses OpenCV Haar cascades

3. **Gaze Tracking**
   - Detects: left, right, up, down, center
   - Uses MediaPipe Face Mesh (468 landmarks)
   - Event type: `gaze_away`

4. **Motion Detection**
   - Detects excessive movement
   - Event type: `high_motion`

5. **Tab Switch Detection**
   - Browser visibilitychange event
   - Event type: `tab_switch`

6. **Input Blocking (Frontend)**
   - File: `hooks/useInputBlocking.js`
   - Blocks: right-click, Ctrl+C/V/X/A, F12, PrintScreen, Ctrl+U

7. **Full-Screen Enforcement**
   - File: `hooks/useFullScreen.js`
   - Auto-prompts on interview start
   - Pauses interview if user exits full-screen mode

**Proctoring Alert Types:**
| Event | Threshold | Action |
|-------|-----------|--------|
| no_face | faces_count == 0 | Warning → Pause after 3 violations |
| multi_face | faces_count > 1 | Warning → Pause |
| face_mimilarity | < 0.70 | Warning → Pause |
| shoulder_missing | score < 0.50 | Warning |
| high_motion | > 0.25 | Warning |
| gaze_away | direction != center | Warning |
| tab_switch | detected | Warning |

### B. VOICE PROCESSING (Whisper)

**Audio Recording:**
- Format: audio/webm (opus codec)
- Chunk size: 1000ms intervals
- Min size check: 2000 bytes (rejects too small)

**Whisper API:**
- Uses: LLM_API_KEY from .env
- Model: whisper-1
- Hallucination detection: Blocks purely URL-like outputs (< 3 words + URL pattern)

**Fallback:** Browser SpeechSynthesis (TTS only)

### C. CANDIDATE FLOW

1. Login/Signup
2. View all active JDs
3. Select JD
4. Upload resume → AI scores against JD
5. If shortlisted → Schedule interview
6. Take interview (questions generated from resume + JD)
7. Submit answers (text or voice)

### D. HR DASHBOARD

**Query Filter (Already Implemented):**
```python
.filter(JobDescription.company_id == hr_id)
```
- Each HR sees ONLY candidates who applied to THEIR jobs
- If candidate applied to multiple HRs' jobs → visible to all respective HRs

**Delete Logic:**
- If candidate has applications to OTHER HRs → only deletes THIS HR's results
- If candidate has ONLY this HR's applications → deletes candidate entirely
